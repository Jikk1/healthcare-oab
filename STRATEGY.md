# Стратегия развития платформы HealthCareOAB+

## 1. Анализ текущего состояния ("As Is")

Проект представляет собой высокотехнологичный фундамент для медицинского SaaS-решения.

*   **Бэкенд:** Модульный монолит на Node.js (Fastify, Prisma). Высокий уровень готовности: реализована многоарендность, шифрование PHI (AES-256-GCM), аудит доступа и система событий (Outbox).
*   **Фронтенд:** Статическое приложение (HTML/JS/CSS). Хорошо подходит для демо, но требует миграции на современный фреймворк для реализации личного кабинета пациента и сложной бизнес-логики.
*   **ИИ-движок (OmniRisk):** Детерминированная модель с высокой объяснимостью. Является отличной отправной точкой, но требует дополнения ML-моделями для повышения точности на больших данных.

---

## 2. Целевая архитектура ("To Be")

Для масштабирования системы до уровня полноценной экосистемы (Врачи + Пациенты + Анализы) предлагается следующая структура:

*   **Core API (Node.js/Fastify):** Центр управления данными, авторизацией и бизнес-логикой.
*   **ML Inference Service (Python/FastAPI):** Исполнение сложных предсказательных моделей.
*   **Integration Layer:** Модули для работы с HL7 FHIR (интеграция с МИС) и парсеры лабораторных отчетов.
*   **Multi-platform Frontend:**
    *   Web (React/Vite) — для врачей и аналитиков.
    *   Mobile (React Native) — для пациентов (мониторинг, рекомендации).

---

## 3. Дорожная карта (Roadmap)

### Фаза 1: Стабилизация и Интеграция (1-2 мес.)
*   **Склейка E2E:** Полный переход дашборда с демо-данных на API.
*   **Миграция на React:** Перенос фронтенда на компонентную архитектуру (Vite + React + Shadcn/UI).
*   **Auth & Security:** Финализация системы JWT Refresh и усиление контроля PHI.

### Фаза 2: Интероперабельность (3-5 мес.)
*   **Модуль "Анализы":** Автоматический импорт и визуализация трендов показателей во времени.
*   **МИС/ЛИС Адаптеры:** Поддержка стандартов обмена данными (FHIR) для интеграции с больницами.
*   **Patient Portal:** Запуск веб-кабинета для пациентов.

### Фаза 3: Интеллектуальный скачок (6-9 мес.)
*   **ML-контур:** Внедрение обученных моделей на базе CatBoost/XGBoost параллельно с текущими формулами.
*   **Новые патологии:** Добавление модулей онкорисков и когнитивного здоровья.
*   **XAI (Explainable AI):** Улучшение визуализации факторов риска (SHAP) для принятия врачебных решений.

### Фаза 4: Экосистема и Экспансия (10-12 мес.)
*   **Mobile App:** Запуск нативных приложений для iOS/Android (мониторинг, цифровой двойник).
*   **Billing:** Финализация платежного модуля и запуск B2B/B2C тарифов.
*   **Compliance:** Прохождение сертификации по медицинским стандартам (152-ФЗ / HIPAA).

---

## 5. Резюме
Проект обладает мощным ядром и готов к быстрому масштабированию. Основной фокус ближайшего времени — **техническая миграция фронтенда** и **автоматизация сбора данных** (интеграция с лабораториями). Это позволит превратить систему из "умного калькулятора" в незаменимый инструмент ежедневной медицинской практики.
